# 风味组报销助手 - 功能改进设计

**日期**: 2026-03-02

## 概述

本文档包含5个功能改进的设计方案：
1. PDF识别功能
2. 电子发票分页展示
3. 管理端按名称模糊搜索
4. 发票卡片查看大图
5. 状态提示居中弹窗

---

## 功能1：PDF识别功能

### 需求
- 支持上传PDF格式的发票
- PDF每页就是一张发票
- 使用阿里云OCR进行识别

### 技术方案

#### 前端改动
- 文件上传组件：接受 `application/pdf` 类型的文件
- 文件预览：使用pdf.js将PDF转为图片进行预览
- API调用：后端需要能处理PDF文件

#### 后端改动
- 修改OCR接口：同时支持image/*和application/pdf
- 阿里云OCR：使用PDF文件直接调用（阿里云通用文字识别支持PDF）

### 实现步骤
1. 前端文件上传组件增加accept属性支持PDF
2. 前端添加PDF预览功能（pdf.js）
3. 后端API接收PDF文件并调用阿里云OCR
4. 阿里云返回结果后解析并返回

---

## 功能2：电子发票分页展示

### 需求
- 电子发票数量过多时使用分页展示
- 格式：1、2、3页码按钮 + 上一页/下一页
- 每页显示6张发票

### 技术方案

#### 前端改动
- 状态管理：添加 `currentPage` 状态
- 分页组件：显示页码按钮和翻页按钮
- 数据过滤：根据当前页码显示对应的发票

#### 数据结构
```typescript
interface PaginationState {
  currentPage: number;
  itemsPerPage: number; // 6
}
```

#### 实现步骤
1. 在发票识别区添加分页状态
2. 创建分页组件（页码按钮 + 翻页按钮）
3. 根据当前页码过滤显示的发票列表
4. 处理空页、首页、末页的边界情况

---

## 功能3：管理端按名称模糊搜索

### 需求
- 管理端可以按用户名称进行模糊搜索
- 不需要支持学号搜索

### 技术方案

#### 前端改动（推荐方案A）
- 添加搜索输入框
- 在前端过滤records数组

#### 实现步骤
1. 在管理端页面添加搜索输入框
2. 添加搜索状态 `searchQuery`
3. 在显示记录时过滤：`name.includes(searchQuery)`
4. 搜索框旁边添加清除按钮

---

## 功能4：发票卡片查看大图

### 需求
- 用户端点击发票卡片可以查看大图
- 弹窗显示发票图片和识别数据

### 技术方案

#### 前端改动
- 状态管理：添加 `selectedRecord` 状态
- Modal弹窗：大图 + 发票数据
- 关闭逻辑：点击外部关闭 或 点击关闭按钮

#### 实现步骤
1. 在用户端报销记录卡片上添加点击事件
2. 创建大图预览Modal组件
3. Modal显示内容：发票图片大图 + 金额 + 发票号 + 销售方等
4. 添加关闭按钮和点击外部关闭逻辑

---

## 功能5：状态提示居中弹窗

### 需求
- 所有状态提示（成功/失败/信息）显示在屏幕正中央
- 弹窗样式：白色弹窗 + 图标 + 文字
- 自动关闭：2-3秒后消失

### 技术方案

#### 前端改动
- 状态管理：复用现有的 `statusMessage` 状态
- Modal组件：居中弹窗 + 半透明遮罩
- 样式：成功绿色、失败红色，信息蓝色

#### 组件结构
```typescript
// 状态提示弹窗组件
<StatusModal
  type="success" | "error" | "info"
  message="删除成功"
  onClose={() => setStatusMessage(null)}
/>
```

#### 实现步骤
1. 创建 `StatusModal` 组件
2. 将现有的顶部状态提示替换为Modal
3. 添加动画效果（淡入淡出）
4. 设置2-3秒自动关闭定时器

---

## 实施顺序

建议按以下顺序实施：

1. **功能5（状态提示）** - 改动最小，先完成
2. **功能4（查看大图）** - UI改动，便于调试
3. **功能1（PDF识别）** - 需要后端配合
4. **功能2（分页展示）** - UI组件开发
5. **功能3（名称搜索）** - 前端过滤，最简单
